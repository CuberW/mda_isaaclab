### 1.3 差异性分析

| 差异点 | 2.2 | 3.7 | 3.19 |
|--------|-----|-----|------|
| **核心难点** | 双臂同步/避碰/姿态保持 | 多相机TF融合/注视切换 | 分类-抓取-投放全流程 |
| **VLA需求** | 强(需理解主从角色) | 中(Pipeline方案可替代) | 弱(分类为主) |
| **协调复杂度** | 极高(双臂实时同步) | 中(单臂分时操作) | 低(单臂顺序操作) |
| **相机数量** | 1-2个 | 3个(头+双腕) | 1-2个 |
| **抓取复杂度** | 协同搬运(姿态保持) | 精细抓取(D405近景) | 通用抓取 |
| **导航需求** | 无 | 无 | 有(轮式移动) |

#### 双臂协调核心算法链路

```
# 典型的双臂协同搬运——以"双手抱起箱子"为例

1. LLM解析: "双手抱起箱子" → {
    双臂角色: "对称协同",
    目标物体: "箱子",
    目标区域: "指定位置",
    约束: "保持箱子水平(roll/pitch<5°)",
    阶段: [approach, grasp, lift, carry, place]
}

2. 子目标生成(DAG):
   ├─ 左臂: move_to(left_grasp_pose)
   ├─ 右臂: move_to(right_grasp_pose)   ← 与左臂并行
   ├─ barrier: 双臂均到达抓取位姿
   ├─ 左臂+右臂: synchronized_lift()    ← 同步执行
   │   约束: |left_ee_pose - right_ee_pose - box_offset| < ε
   ├─ 左臂+右臂: synchronized_carry(target_region)
   └─ 左臂+右臂: synchronized_place()

3. 轨迹生成(WBC):
   - 输入: 左右臂目标位姿 + 箱子姿态约束
   - 全身IK求解 (joint velocity control)
   - 碰撞检测 (自碰 + 环境碰)
   - 平滑插值 (cubic spline)

4. VLA闭环微调:
   - 在线观测物体姿态偏差
   - 修正两臂末端位姿
   - 保证协同搬运质量
```

### 8.2 重点系统深度分析

#### 8.2.1 CogACT (Microsoft, 2024.11) ⭐推荐借鉴

```
┌─────────────────────────────────────────────────────────────┐
│                    CogACT 架构                                │
│                                                             │
│  ┌──────────────────────┐    ┌──────────────────────────┐   │
│  │  VLM (认知模块)       │    │  DiT (动作模块)           │   │
│  │  ├─ SigLIP (视觉)    │───▶│  ├─ Diffusion Transformer│   │
│  │  ├─ DINOv2 (视觉)    │    │  ├─ 16步动作序列预测      │   │
│  │  └─ Llama2-7B (语言) │    │  └─ 自适应动作集成(AAE)   │   │
│  └──────────────────────┘    └────────────┬─────────────┘   │
│                                           │                 │
│  输入: 单张RGB + 语言指令                   │                 │
│                                       7-DoF动作              │
│  输出: x,y,z,roll,pitch,yaw,gripper       │                 │
│  速度: ~181ms/推理 (A6000)  ~5.5Hz        │                 │
│  性能: 超OpenVLA 35% (仿真) 55% (真机)     │                 │
└─────────────────────────────────────────────┘
```

**架构借鉴点**：
1. **认知-动作分离设计**（Cognition-Action Decoupling）：VLM负责语义理解，DiT专注动作生成——比OpenVLA的单一VLM更模块化
2. **自适应动作集成**（Adaptive Action Ensemble）：对扩散模型的多步采样结果加权平均，提升动作质量——可直接用于2.2的双臂同步
3. **组件可替换性**：VLM可换为Qwen2.5-VL，Action Module可换为更小的DiT-S→适配RTX 5070Ti 12GB
4. **MIT许可**：可自由修改和集成

**与我们的任务映射**：
- 2.2: 将单臂DiT扩展为双臂DiT（输出14-DoF），VLM改为Qwen2.5处理中文指令
- 3.7: 保留单臂DiT，VLM增加多图输入（头+腕）
- 3.19: 用分类器替代VLM的开放性，DiT简化为抓取位姿预测

#### 8.2.2 AgiBot World GO-1 (智元+上海AI Lab, 2025.03) ⭐推荐借鉴

```

│                  GO-1 ViLLA 架构                              │
│                                                             │
```

### 9.0 关键技术决策说明

#### 决策 1：单目深度估计方案选型

在真机Dobot场景中无可用的RGB-D相机，需通过单目方案获取深度信息。

**可选方案对比**：

| 方案 | 精度 | 延迟 | 部署难度 | 适用场景 |
|------|------|------|----------|----------|
| **Depth Anything v2** (推荐) | 高(metric) | ~50ms | pip install即用 | 通用桌面抓取 |
| **Apple Depth Pro** | 高(zero-shot) | ~100ms | 需Apple MLX | 复杂场景 |
| 射线-平面几何求交 | 低(粗略) | 实时 | 纯数学计算 | 已知平面+非堆叠物体 |

Depth Anything v2 是2024年单目深度估计SOTA，MIT许可。学术验证表明单目深度+抓取检测组合在桌面场景中可达93.2%真机成功率（DepthAugmentedGraspNet, ACM 2024）。

射线-平面求交法仅在物体明显置于已知桌面且仅需粗糙3D位置时作为快速fallback，不能处理堆叠物体和复杂几何形状。

**采用策略**：仿真中用MuJoCo直接渲染深度图（精度完美），真机中用Depth Anything v2进行单目深度估计。

#### 决策 2：Kuavo仿真环境获取路径

Kuavo仿真环境由乐聚机器人官方维护，通过Gitee仓库发布。

**调研结论**：
- Kuavo官方文档（kuavo.lejurobot.com）活跃更新至2025年11月（v1.3版本，新增MuJoCo灵巧手控制+VR遥操作）
- GitHub社区镜像 `stjordanis/kuavo_opensource` 可用，降低Gitee网络依赖
- Docker镜像通过 `./docker/run_with_gpu.sh kuavo_opensource_mpc_wbc_img:0.6.1` 脚本构建
- 第三方项目（SH9959/ZT、Ant Group LingBot-VLA）已验证仓库可编译性

**获取路径**：
- **路径A**（推荐）：Gitee clone源码 → `./docker/run_with_gpu.sh` 本地构建（ROS Noetic + GPU）
- **路径B**：GitHub社区镜像 clone（避免Gitee网络问题）
- **路径C**（兜底）：robosuite v1.5自定义人形场景

预估环境搭建耗时约2天。

#### 决策 3：3.19机器人平台选型（Stretch vs GR-1 vs 自定义）

**候选平台对比**：

| 维度 | Stretch | GR-1 (傅利叶) | 自定义MuJoCo |
|------|---------|---------------|-------------|
| 形态 | 轮式+伸缩臂 | 双足人形 | 完全可控 |
| 匹配作业需求 | ✅ 轮式操作 | ❌ 非轮式 | ✅ 可定制 |
| MuJoCo官方仿真 | ✅ hello-robot维护 | ❌ 需Isaac Sim | 需自行搭建 |
| ROS2支持 | ✅ stretch_ros2 | 有限 | 需自行实现 |
| 传感器 | D405 RGB-D + LiDAR | RGB相机阵列 | 需自行配置 |
| 部署难度 | pip install即用 | 高 | 高 |

GR-1经傅利叶官方规格书（fftai.cn）确认：身高1650mm，6-DoF×2腿，步态行走，5km/h——是纯双足人形，无轮式配置。不适用于3.19作业的轮式人形需求。

**采用**：Stretch（hello-robot/stretch_mujoco），轮式差分驱动+4-DoF伸缩臂，MuJoCo官方仿真，跨平台pip安装。

#### 决策 4：Kuavo仿真接口的真实性

Kuavo官方文档描述的ROS话题接口（`/joint_cmd`、`/kuavo_arm_traj` 等）在仿真和真机间保持一致。这是机器人SDK的标准设计——同一套ROS接口，底层driver不同。

仿真与真机之间客观存在sim-to-real gap（动力学差异、传感器噪声等），接口一致不代表行为一致。域随机化训练和噪声增强是后续真机迁移时的必要手段。

### 9.2 2.2 双臂VLA→Dobot Magician 迁移分析

#### 可行性评估

| 维度 | 仿真(Panda) | 真机(Dobot) | 差距 | 迁移策略 |
|------|------------|-------------|------|----------|
| DoF | 7-DoF × 2 | 4-DoF × 2 (多台已有) | 每臂少3-DoF | 仅用位置控制(x,y,z,r)，放弃姿态全约束 |
| 工作空间 | ~855mm半径 | ~320mm半径 | 缩小62% | 缩小任务场景比例 |
| 载荷 | ~3kg | ~250g | 差12倍 | 用轻质物体(纸杯/泡沫块) |
| 末端工具 | 平行夹爪 | 气泵吸盘 | 不同 | 物体需有平面可吸附 |
| 双臂配置 | 出厂双臂 | 桌面并列放置 | 需自行标定 | 桌面并列放置两台Dobot，标定基座相对位姿 |
| 控制接口 | libfranka | pydobot(串口) | 协议不同 | 统一Python API包装层 |
| SDK成熟度 | 极高 | 社区维护 | 稳定性存疑 | magician_ros2作为ROS层 |

**结论：可行性高（⭐⭐⭐⭐）**。多台Dobot Magician已具备，消除了最大的硬件障碍。

核心挑战和应对：
1. ✅ **多台Dobot已具备**——可直接搭建桌面双臂实验平台，参考 chess_system_dobots_arms / DobotCityBuilding 等项目的双Dobot协调方案
2. ⚠️ 4-DoF 无法实现完整的6-DoF姿态保持（roll/pitch不可控）——通过任务设计规避：选择**平面内搬运**任务（物体自然水平放置），或使用**顶抓+平移**策略（不涉及旋转）
3. ⚠️ 工作空间仅320mm半径——缩小任务场景，使用轻质小物体
4. ⚠️ 末端为气泵吸盘（非夹爪）——物体需有平面可吸附；可打印3D夹爪改装件
5. ✅ 双Dobot协调已有成熟开源参考（chess_system、DobotCityBuilding等）

### 9.3 3.7 Kuavo→Dobot Magician 迁移分析

#### 可行性评估

| 维度 | 仿真(Kuavo) | 真机(Dobot) | 迁移策略 |
|------|------------|-------------|----------|
| 头部相机 | Gemini-335L RGB-D | USB摄像头 (2D only) | 用Depth Anything v2(MIT许可)单目深度估计；仅在平坦桌面做fallback时用射线-平面几何近似 |
| 腕部相机 | D405 RGB-D × 2 | 固定USB摄像头 (2D) | 简化为固定俯视相机，不需要注视切换 |
| 双手臂 | 人形双臂(6-DoF×2) | 双Dobot(4-DoF×2) | 多台Dobot可用，双臂协作可行 | 保留双臂优势，简化为4-DoF顶抓 |
| 开放词汇检测 | 头部广角→腕部近景 | 固定双Dobot视角 | 双Dobot各配USB相机 | 扩展为"双Dobot多视角"替代注视切换 |
| 抓取姿态 | 6-DoF | 4-DoF(自上而下) | 简化为顶抓(top-down grasp) |

**结论：可行性高（⭐⭐⭐⭐⭐）**。Kuavo→多Dobot的迁移是三任务中可行性最高的：

1. 3.7的核心是**开放词汇感知+抓取**，双臂不是必选项——单臂或双臂均可演示
2. **多台Dobot可用**：可以用双Dobot模拟Kuavo的双臂场景（左臂固定物体、右臂抓取），甚至比单臂更接近原场景
3. 相机简化：可为每台Dobot配备USB相机，实现"多视角"而非"注视切换"——架构理念一致
4. 开放词汇抓取Pipeline（Grounded-SAM+AnyGrasp→Dobot）架构清晰可落地；单目深度用Depth Anything v2（2024 SOTA，MIT许可）替代RGB-D相机，学术界已验证该路线在桌面抓取场景的可行性（DepthAugmentedGraspNet: 93.2%真机成功率）

| **文档、代码与视频** | 15 | 完整闭环演示 + 失败案例分析 | ⚠️ 视频展现从"初始凌乱场景"到"全部垃圾分类投放完毕" | 10+物体整体处理时间 |

**得分冲刺建议**：
- 干扰样本(20分)是独特得分点：准备5+干扰样本(可回收物上贴食物残渣、变形塑料瓶等)，对比YOLOv8 vs YOLOv8+CLIP的鲁棒性
- 全流程闭环视频：一镜到底展示扫描→识别→抓取→移动→投放→下一个→直到清空
- 统计表格式美观：用表格展示每一件垃圾的(原始类别, YOLO预测, CLIP确认, 投放桶, 是否正确)
